# VS Code Keyboard Shortcuts — Example

## Scenario: Onboarding a Frontend Developer Who Still Uses Menus

**Who:** Priya Nair, Junior Frontend Developer, 6 months into her first role at a SaaS startup.
**Where:** VS Code on macOS, working on a React + TypeScript codebase.
**The Problem:** Priya navigates almost entirely by mouse: clicking File > Open, right-clicking to find commands, and scrolling to find functions. In code reviews, her senior colleagues mention she should "speed up" but haven't given her a concrete starting point. She spends more time navigating than writing code.

**Prerequisites:** VS Code 1.85+, any TypeScript or JavaScript project open.

---

### What You Say to the Agent

> "I'm a junior developer and I still use the mouse for most things in VS Code. What shortcuts should I learn first to actually speed up my workflow?"

### What the Agent Does

1. Prioritizes the three highest-leverage shortcuts for a beginner: Command Palette (`Cmd+Shift+P`), Quick Open (`Cmd+P`), and multi-cursor with `Cmd+D`.
2. Explains the Command Palette as a replacement for menu navigation, with the `>`, `@`, and `#` prefix syntax.
3. Walks through Quick Open with the `filename:linenumber` pattern and how to cycle recent files.
4. Demonstrates `Cmd+D` for selecting next occurrence and `Cmd+Shift+L` for all occurrences, with a realistic use case (renaming a component prop across a JSX file).
5. Shows how to open and search the Keybindings editor with `Cmd+K Cmd+S` for discovering additional shortcuts.
6. Provides the Quick Reference table so Priya can keep it open while she practices.

### The Business Impact

- **Onboarding speed:** Priya reduces navigation time by an estimated 30-40%, freeing cognitive load for the actual code.
- **Code review feedback:** Senior reviewers see faster iteration in pair programming sessions.
- **Confidence:** Having a clear list of 10 shortcuts to master removes the overwhelm of "there are too many shortcuts to learn."

# Content Engine — Example

## Scenario: Repurposing a YouTube Video on MCP for D365 Across Three Platforms

**Who:** Ragnar Pitla, content creator and AI educator, publishing across YouTube, LinkedIn, and X
**Where:** Post-publish workflow after a YouTube video goes live on @RagnarPitla
**The Problem:** Ragnar published a 14-minute YouTube video explaining how MCP works with D365 Finance and Operations. The core idea is strong: MCP turns D365 into a tool layer that any AI agent can call, without custom connectors per agent. But he needs a LinkedIn post, a newsletter section, and three X threads that each feel native to their platform, not like copy-pasted captions under a link. He does not have time to rewrite from scratch for each channel.

**Prerequisites:** YouTube video title, core argument (one sentence), and the target platforms. The video transcript helps but is optional.

---

### What You Say to the Agent

> "I just published a YouTube video: 'How MCP Connects AI Agents to D365 Without Custom Connectors.' Core idea: MCP is the universal adapter layer — instead of building a custom connector for every AI agent that needs to talk to D365, you build one MCP server and every agent can use it. Repurpose this into: a LinkedIn post (150-200 words, native format, no hashtag dump), a newsletter section (250 words, first-person, framework-focused), and 3 X threads (each 4-5 tweets, different angles, no 'check out my video' opener)."

### What the Agent Does

Produces four platform-native pieces from one brief:

**LinkedIn post** — opens with a contrarian statement ("Your D365 integration problem is not a D365 problem"), builds to the MCP insight, ends with a concrete takeaway and one link. Short paragraphs. No wall of text.

**Newsletter section** — written in Ragnar's first-person voice, framed as a framework lesson. Includes a named concept ("the adapter tax"), a three-step explanation of how MCP eliminates it, and a callout box with the key principle.

**X Thread 1** — angle: the business case for MCP. Opens with a stat or provocation, walks through the problem, ends with a clear call to action.

**X Thread 2** — angle: technical explainer for developers. Shows a before/after of connector architecture.

**X Thread 3** — angle: prediction/hot take. "In 18 months, every enterprise AI agent will use MCP or a protocol like it. Here is why resistance is futile."

### The Business Impact

- **One video becomes five assets** — distributed across three platforms without each piece feeling like the same idea repeated
- **Platform-native feel** — LinkedIn gets the framework framing, X gets the hot take energy, newsletter gets the depth
- **Total production time under 20 minutes** — versus a half-day of context-switching between platforms and rewriting

### Try It Yourself

> "Same video. Now generate a short-form script (60-90 seconds) for a YouTube Short or TikTok clip. Pull the single most surprising insight from the video and build the short around it. Hook in the first three words."
